# 旅行方案验证技能

对生成的旅行方案进行全面审查，确保方案可执行且合理。

## 适用场景

- 完整的旅行方案已经生成，需要进行质量审查
- 检查行程安排中的逻辑错误和不合理之处
- 对比预算估算与实际价格的偏差

## 验证维度

### 1. 完整性检查

- 是否涵盖每一天的行程安排
- 是否包含住宿信息
- 是否包含交通方案
- 是否包含预算明细
- 是否包含实用信息（签证、货币、安全等）

### 2. 时间合理性

- 每日行程是否过于紧凑（超过 12 小时活动）
- 景点间交通时间是否合理
- 是否考虑了开放时间和排队时间
- 第一天和最后一天是否考虑了抵达/离开时间

### 3. 预算一致性

- 各项费用加总是否等于总预算
- 费用估算是否在合理范围内
- 是否超出用户预算限制

### 4. 逻辑一致性

- 地理路线是否合理（避免来回折返）
- 住宿位置与行程安排是否匹配
- 餐饮安排与行程路线是否协调

### 5. 实用性审核

- 推荐的交通方式是否可行
- 景点是否在旅行日期内开放
- 是否有季节性或节假日影响

## 输出格式

```json
{
  "overall_score": 85,
  "status": "pass",
  "issues": [
    {
      "severity": "warning",
      "category": "时间安排",
      "detail": "第3天行程安排了6个景点，建议减少到3-4个",
      "suggestion": "将故宫和天坛安排在同一天，长城单独一天"
    }
  ],
  "summary": "方案整体合理，存在2个警告需关注"
}
```

## 评分标准

| 分数   | 状态               | 说明                         |
| ------ | ------------------ | ---------------------------- |
| 90-100 | pass               | 方案优秀，可直接使用         |
| 70-89  | pass_with_warnings | 方案可用，但有改进空间       |
| 50-69  | needs_revision     | 存在较多问题，需要修改       |
| 0-49   | fail               | 方案存在严重问题，需重新规划 |
